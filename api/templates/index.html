Neural Works Test

Esta API proporciona una serie de endpoints y vistas que te permiten interactuar con la aplicación y acceder a las funcionalidades.
A continuación, encontrarás una descripción de cada uno de los endpoints disponibles.

DISCLAIMER: Esta simple aplicación por temas de tiempo no tiene un manejo de errores implementado y/o casos bordes por ingresar mal los inputs,
por lo que se sugiere ingresar datos en los forms como indican, o enviar datos en json en el formato esperado.

Crear Viaje

Este endpoint permite crear un nuevo registro de viaje en la base de datos. Puede recibir datos en formato JSON a la ruta /create_trip o mediante un formulario HTML en la vista Crear Viaje. Utiliza el método POST para procesar la solicitud.

Lista de Viajes

Este endpoint proporciona una lista de los último 25 viajes ingresados con un método GET a la ruta /trips.

Viajes Agrupados

Este endpoint proporciona un resumen de los viajes agrupados por región y períodos de tiempo de 6 horas a lo largo de un día. Utiliza el método GET para recuperar los datos en la ruta /grouped_trips.

Promedio Semanal de Viajes

Este endpoint calcula el promedio semanal de la cantidad de viajes para un área definida por un bounding box y la región. Puede recibir datos en formato JSON o mediante un formulario HTML. Utiliza los métodos GET y POST para procesar la solicitud en la ruta /average_trips

Cargar Datos

Este endpoint no tiene interfaz, se utiliza para cargar datos desde un archivo CSV en la base de datos con el método GET en la ruta /load_data. Asegúrate de tener un archivo CSV válido, en este caso se inserto `trips.csv` en la raíz del proyecto antes de usar este endpoint. Disclaimer: cargar un archivo mas de una vez sumará el doble de datos.

Informar Estado de Ingesta de Datos

El endpoint /notifications se utiliza para recibir sobre el estado de la ingesta de datos. Utiliza el método POST para enviar información desde /create_trips a /notifications cada vez se ingresa un viaje.
